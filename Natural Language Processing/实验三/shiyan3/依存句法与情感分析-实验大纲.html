实验大纲：基于依存句法树的中文方面级情感分析

实验目的

  1. 掌握依存句法分析的基础理论及其在自然语言处理中的应用。学习HanLP工具使用。
  2. 理解方面级情感分析（Aspect-Based Sentiment Analysis, ABSA）的任务目标与流程。
  3. 设计并实现一套基于依存句法树的规则方法，用于自动无监督地从文本中抽取方面词（Aspect Terms）和情感词（Opinion Words）。
  4. 评估所提方法在特定中文方面级情感数据集上的性能。

实验环境与准备

  • 软件环境：Python 3.x, Jupyter Notebook或其他Python IDE。
  • 依赖库：jieba, hanlp, pandas等。
  • 数据集：中文方面级情感数据集，sentiment.tsv。

实验步骤

1. 数据预处理与依存句法分析

  • 任务：对数据集进行分词处理，并使用依存句法分析工具对每个句子构建依存句法树。
  • 操作步骤：
    • 使用hanlp对文本进行分词。
    • hanlp工具，对分词后的句子进行依存关系标注，获取依存句法树。
    • hanlp工具安装：pip install hanlp (https://github.com/hankcs/HanLP)
    • 给出这句话的依存句法分析结果：“阿婆主来到北京立方庭参观自然语义科技公司。”
    • 要求打印上述句子依存句法分析结果，并在实验报告中解释哪个词是哪个词的子节点，每个边对应的句法标签的含义是什么。如果能以图的形式画出句法树可以额外加分。
    • 依存句法标签的说明参考：依存句法分析标签说明.md

2. 设计句法规则提取方面词与情感词

  • 任务：基于依存句法关系，设计规则来识别句子中的方面词和情感词。
  • 句法规则:
    • 方面词提取规则（依存句法标签的说明参考：依存句法分析标签说明.md）：
    • 提取"nsubj"(名词主语)关系中的名词作为候选方面词。例如:"房间(nn) 很(advmod) 脏(adj)"中"房间"是"脏"的nsubj,因此"房间"是方面词。
    • 提取"dobj"(直接宾语)关系中的名词作为候选方面词。例如:"我(nsubj) 喜欢(VV) 这个(det) 外观(dobj)"中,"外观"是"喜欢"的dobj,因此"外观"是方面词。
    • 提取"nmod:topic"(主题修饰语)关系中的名词作为候选方面词。例如:"这台机(nmod:topic) 性能(nn) 不- 错(adj)"中,"这台机"是"性能"的nmod:topic,因此"这台机"是方面词。
    • 提取"amod"(形容词修饰语)关系中的形容词作为候选情感词。例如:"外观(dobj) 十分(advmod) 好(amod)- "中,"好"是"外观"的amod,因此"好"是情感词。
    • 提取"advmod"(副词修饰语)关系中的副词作为程度修饰词,与其修饰的形容词一起作为情感词。例如:"性能(nn) 非常(advmod) 强大(amod)"中, "非常"和"强大"一起作为情感词。
    • 提取"conj"(连接词)关系,如果连接词连接两个形容词,则将这两个形容词都作为情感词。例如:"酒店(nn) 服务(nn) 热情(adj) 而且(cc) 细致(conj)"中,"热情"和"细致"都是情感词。
    • 提取"xcomp"(补语)关系中的形容词作为情感词,其所修饰的名词作为方面词。例如:"房间(nsubj) 显得(VV) 很(advmod) 狭窄(xcomp)"中,"房间"是方面词,"狭窄"是情感词。
    • 提取"ccomp"(从句补语)关系,如果从句中包含情感词,则将从句的主语作为方面词。例如:"我(nsubj) 觉得(VV) 这个(det) 地毯(nsubj) 很(advmod) 脏(ccomp)"中,"地毯"是方面词,"脏"是情感词。
    • 对于"neg"(否定关系),如果情感词前有否定词,则将情感倾向反转。例如:"这个(det) 服务(nn) 不(neg) 太(advmod) 好(amod)"中,"不太好"表达负面情感。
    • 对于一些常见的表示情感的动词,如"喜欢"、"讨厌"、"满意"等,将其视为情感词,并将其主语或宾语作为方面词。

3. 实现规则匹配与抽取

  • 任务：编写代码实现上述规则，遍历依存句法树，识别并抽取方面词与情感词。
  • 操作步骤：
    • 遍历每个句子的依存树，检查节点之间的依存关系是否满足预定义的规则。
    • 根据规则匹配情况，标记出潜在的方面词和情感词。

4. 结果评估与分析

  • 任务：评估抽取的方面词和情感词的准确性，对比人工标注数据。
  • 操作步骤：
    • 人工检查抽取结果的一部分，随机抽取十个句子判定抽取结果是否符合预期。
    • 分析错误案例，寻找规则上的遗漏或误判原因。
    • 调整规则或增加新的规则以改进性能。

实验报告要求

  • 方法描述：详细说明实现流程。
  • 结果展示：展示抽取的方面词与情感词实例，对比人工标注结果。
  • 性能评估：给出随机的十个句子以及对应方面词和情感词。

HanLP工具使用示例

import hanlp
HanLP = hanlp.load(hanlp.pretrained.mtl.CLOSE_TOK_POS_NER_SRL_DEP_SDP_CON_ELECTRA_SMALL_ZH)
results = HanLP(['人咬了狗', '阿婆主来到北京立方庭参观自然语义科技公司。'])

def process_dep(dep_list):
    # 把依存分析结果简单处理一下，方便阅读和后续使用
    new_dep_list = []
    for dep_sent in dep_list:
        new_dep_list.append([(tup[0] - 1, tup[1]) for tup in dep_sent ])
    return new_dep_list

print('分词后的结果: ', results["tok/fine"])
print('词性分析结果: ', results["pos/pku"])
print('依存句法分析结果: ', process_dep(results["dep"]))
'''
针对第一个句子我们分析其依存句法树的结果：
原句：人咬了狗
分词：    ['人', '咬', '了', '狗']
依存分析： [(1, 'nsubj'), (-1, 'root'), (1, 'asp'), (1, 'dobj')]

可以看到分词后的列表长度和依存分析结果列表长度一致，
接下来我们这样看，把分词和句法分析结果一一对应：
'人', (1, 'nsubj'), 
'咬', (-1, 'root'), 
'了', (1, 'asp'), 
'狗'，(1, 'dobj')
这里，'人'对应的数字是1，表示'人'这个单词的父节点是句子中第1个单词（从0开始），也就是说'人'的父节点是'咬'，'nsubj'则表示'人'是'咬'
这个动词的主语；
然后，'咬'对应数字是-1，表示'咬'就是根节点，没有父节点，这个边的类型是'root'；
同样的，'了'的父节点是1，即也是'咬'，边的类型是'asp'（"aspect marker"，即体标记。它用于表示一个动词或谓语的时态、时制、体貌等语法范畴）
最后'狗'的父节点是1，边的类型是'dobj'，表示'狗'是动作'咬'的宾语
'''
